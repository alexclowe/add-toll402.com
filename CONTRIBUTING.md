# Contributing to Awesome MCP Monetization

Thank you for your interest in contributing! This list aims to be the definitive resource for MCP server monetization.

## Guidelines

### Adding a New Entry

1. Make sure the resource is related to MCP server monetization.
2. Check that the resource isn't already listed.
3. Use the following format:

```markdown
- [Name](https://link) — Brief, factual description of what it does. No marketing language.
```

4. Add the entry to the appropriate section, in alphabetical order within that section.
5. Make sure all links are working.

### Quality Standards

- **Factual descriptions only** — No superlatives, marketing language, or subjective claims.
- **Working links** — All URLs must resolve to live pages.
- **Relevant to MCP monetization** — The resource should directly relate to monetizing MCP servers or tools.
- **Open source preferred** — For SDKs and libraries, open-source projects are preferred. Commercial platforms are listed in the Platforms & Services section.

### What We Accept

- Payment protocols and standards applicable to MCP
- SDKs, libraries, and middleware for MCP payments
- Platforms and services for MCP monetization
- Registries with monetization features
- Tutorials, guides, and documentation
- Boilerplate code and starter templates
- Pricing strategy resources specific to API/tool monetization

### What We Don't Accept

- General payment processing tools with no MCP relevance
- Resources behind paywalls (tutorials/guides should be freely accessible)
- Abandoned projects (no commits in 12+ months with open issues)
- Duplicate entries or resources that overlap significantly with existing entries

## Process

1. Fork this repository.
2. Create a new branch: `git checkout -b add-resource-name`
3. Make your changes following the guidelines above.
4. Submit a pull request with a clear description of what you're adding and why.

## Updating Existing Entries

If a project has changed significantly (renamed, deprecated, new features), please submit a PR to update the entry. Include a brief note about what changed.

## Reporting Issues

If you find broken links, inaccurate descriptions, or have suggestions for new sections, please [open an issue](https://github.com/xpaysh/awesome-mcp-monetization/issues).

## Code of Conduct

Be respectful and constructive. This is a community resource — contributions should benefit everyone in the MCP ecosystem.
